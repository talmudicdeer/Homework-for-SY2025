**What is a queue?**
Queue is an interface that can be used to add and/or remove elements from the operation.
A FIFO, or first-in-first-out, queue operates like a queue of humans, removing elements in
the order in which they were added; the first in line is the first one out. A **priority-type
queue** removes the smallest element, and arbitrarily breaks ties. Morin uses the example
of the order in which emergency room patients are triaged for the priority-type queue. The
third type is a LIFO, or last-in-first-out, queue. This works, as Morin explains, like a stack of
plates; the last one added is the first one off the stack. This format is so common that it’s
usually just called that: the **stack,** although add(x) and remove(x) are renamed to push(x)
and pop(x) to avoid confusion. When generalized into a single deal, the FIFO queue and
stack are called a **deque,** with some added operations for element management.

**Define USet and SSet.**
They’re both types of the Set interface, but differ in their form and operation. USet is a set
of unsorted elements, and SSet is a set of sorted elements. The find(x) method, which is
used to locate (x) in a sorted set, is what sets the two apart.

**Define Abstract Data Type (ADT).**
ADT sets out rules for behavior for a data set. It doesn’t dictate implementation.
